# МагазинТехники (1С:Магазин техники)

Конфигурация 1С:Предприятие 8.3 — упрощённая учётная система для небольшого магазина техники: продажи и закупки товаров, управление остатками в разрезе складов, ведение базы клиентов и ценообразование.

Исходники выгружены в XML-формате штатным механизмом платформы и версионируются через git.

## О проекте

Репозиторий содержит исходники конфигурации 1С:Предприятие, выгруженные через **Конфигурация → Выгрузить конфигурацию в файлы...**. Такой формат позволяет вести историю изменений в git, делать code review по XML-диффам и работает даже на базовых версиях платформы без хранилища конфигурации.

| Параметр | Значение |
|---|---|
| Внутреннее имя | `МагазинТехники` |
| Синоним | Магазин техники |
| Режим запуска по умолчанию | Управляемое приложение |

| Язык по умолчанию | Русский |

## Состав конфигурации

### Справочники

| Справочник | Назначение | Реквизиты |
|---|---|---|
| `Товары` | Номенклатура магазина | `Артикул`, `ЕдиницаИзмерения`, `Цена`, `Картинка` |
| `Клиенты` | База покупателей | `ФИО`, `Адрес`, `Телефон` |
| `Склады` | Места хранения товаров | — (только наименование) |
| `ЕдиницаИзмерения` | Единицы измерения товаров (шт., кг., упак. и т.д.) | — (только наименование) |

### Документы

**ПродажаТоваров** — фиксирует факт продажи товаров покупателю.
- Шапка: `Клиент`
- Табличная часть «Товары»: `Товар`, `Склад`, `Количество`, `Цена`, `Сумма`
- При проведении формирует движения по регистру `ОстаткиТоваров` (расход)

**ПоступлениеТоваров** — фиксирует приёмку товаров на склад.
- Табличная часть «Товары»: `Товар`, `Склад`, `Количество`, `Цена`, `Сумма`
- При проведении формирует движения по регистру `ОстаткиТоваров` (приход)

### Регистры

| Регистр | Тип | Измерения | Ресурсы |
|---|---|---|---|
| `ОстаткиТоваров` | Накопления (остатки) | `Товар`, `Склад` | `Количество`, `Цена`, `Сумма` |

### Обработки

| Обработка | Назначение |
|---|---|
| `ИзменениеЦенТоваров` | Массовое изменение цен на товары |

### Общие модули

| Модуль | Контекст | Назначение |
|---|---|---|
| `ОбщиеМодулиДокументов` | Клиент + Сервер | Общая логика проведения документов |

## Бизнес-процессы

**Продажи:**
```
ПродажаТоваров (проведение)
       │
       └─→ AccumulationRegister.ОстаткиТоваров (расход остатка по складу)
```

**Закупки:**
```
ПоступлениеТоваров (проведение)
       │
       └─→ AccumulationRegister.ОстаткиТоваров (приход остатка по складу)
```

**Ценообразование:**
```
ИзменениеЦенТоваров (обработка)
       │
       └─→ Catalog.Товары (обновление поля «Цена»)
```

## Структура репозитория

```
.
├── Configuration.xml              # корневое описание конфигурации
├── ConfigDumpInfo.xml             # служебная информация о выгрузке

├── Catalogs/
│   ├── Товары.xml
│   ├── Клиенты.xml
│   ├── Склады.xml
│   └── ЕдиницаИзмерения.xml

├── Documents/
│   ├── ПродажаТоваров.xml
│   └── ПоступлениеТоваров.xml

├── AccumulationRegisters/
│   └── ОстаткиТоваров.xml

├── DataProcessors/
│   └── ИзменениеЦенТоваров.xml

└── CommonModules/
    └── ОбщиеМодулиДокументов.xml
```

> Реальная раскладка файлов соответствует штатной структуре выгрузки конфигуратора; каждый объект хранится в своей подпапке с `Ext/`.

## Требования

- Платформа 1С:Предприятие 8.3 (версия 8.3.27 и выше)

## Обновление репозитория после изменений конфигурации

1. Откройте базу в режиме **Конфигуратор**.
2. Выполните **Конфигурация → Выгрузить конфигурацию в файлы...**, указав папку репозитория — изменятся только реально изменённые файлы.
3. Зафиксируйте изменения в git:

```bash
git add .
git commit -m "Описание изменений"
git push
```

## Развёртывание конфигурации из репозитория

1. Создайте новую информационную базу в 1С (или используйте существующую).
2. В Конфигураторе выполните **Конфигурация → Загрузить конфигурацию из файлов...**, указав путь к папке репозитория.
3. Обновите конфигурацию базы данных (**F7**).
